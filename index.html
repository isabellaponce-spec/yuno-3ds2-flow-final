3DS2 — Receiving the device fingerprint when Yuno's checkout is not used

Yuno's existing 3DS flow assumes Yuno's checkout / SDK collects the device fingerprint. When the merchant uses their own checkout (Zuora HPM, a Zuora iframe, the merchant's own front-end), Yuno cannot collect it directly. We need a way for the merchant to pass the fingerprint to Yuno on the V1 Payments request.

The single addition to the public docs: a new inbound field on POST /v1/payments where the merchant passes the device fingerprint they collected on their own checkout. Field name and exact location owned by the Core team's spike.

What stays the same

  • Response object payment_method.detail.card.card_data.three_d_secure — unchanged.
  • redirect_url location in payment.payment_method.payment_method_detail.card — unchanged.
  • Status flow CREATED → PENDING → IN_PROCESS → SUCCEEDED / DECLINED / ERROR — unchanged.
  • Webhook + GET /v1/payments/{id} — unchanged.

Ownership

Core team. The field lives on V1 Payments, which Core owns.
